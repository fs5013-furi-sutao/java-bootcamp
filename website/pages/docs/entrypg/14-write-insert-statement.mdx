# 14. INSERT文の書き方
データベースにレコードを追加する方法を説明する

## 基本的な書き方

テーブルの値を更新する場合は、INSERT文を使います。

INSERT文を使う場合は値を入れるカラム名の一覧とそこに入れる値を記述しました。

ここで`値1`は`カラム1`に、`値2`は`カラム2`に入りますので、順番を揃える必要があります。

```sql
INSERT INTO テーブル名 (カラム名1, カラム名2...)
VALUES (値1, 値2...);
```

### 1.テーブルにレコードを追加する

```sql
INSERT INTO orders (orderNumber, orderDate, requiredDate, status, customerNumber)
VALUES (40099, "2023/10/26", "2023/11/5", "手配中",8103);
```

### 2.必須項目とは
上のINSERT文で注目したいのは、`shippedDate`および`comments`カラムに値を入れていないことです。
同じように`customerNumber`に値を入れないINSERT文を実行するとどうなるでしょうか。

```sql
INSERT INTO orders (orderNumber, orderDate, requiredDate, status)
VALUES (40098, "2023/10/26", "2023/11/5", "手配中");
```

この場合はエラーが発生します。エラーの原因は`customerNumber`が必須項目だからです。
テーブルを作成する際は、必ず値を入れなければいけないカラムを決めることができます。

INSERT文にレコードを追加する場合は、必須項目となっているカラムに対して必ず値を入れなければいけません。

必須項目となっているカラムはA5 SQL:Mk-2にて確認することができます。
テーブルを開いて、`カラム`タブでクリックするとテーブルのカラム一覧が表示されます。

カラム一覧のなかで`必須`項目が`Yes`となっているものが必須項目です。

![テーブル追加後](./insert-not-null.png)

ここで`必須`項目以外にも`データ型`という項目が確認ができます。

テーブルにレコードを追加する場合は、データ型を意識する必要もあります。

### 3.カラムのデータ型

Javaのデータ型のようにMySQLにもデータ型が存在します。

代表的には数値を入れるための`INT型`や文字列を入れるための`VARCHAR型`が存在しますが、ほかにもいくつか存在しますので、詳しく知りたい方はWebで調べてください。検索ワードは、「 [MySQL データ型](https://www.google.com/search?q=MySQL+データ型) 」です。
