# 02. ブラックボックステスト/ホワイトボックステスト

## 学習

### ブラックボックステストとホワイトボックステスト

テストには大きく分けて2種類のアプローチが存在します。

プログラム内部の構造を意識しないブラックボックステストとプログラムの内部構造を意識して、正しく動作するかを確認するホワイトボックステストがあります。
本試験ではホワイトボックステストを行います。

詳しくはWebで調べてください。検索ワードは、「 [ブラックボックステスト　ホワイトボックステスト　違い](https://www.google.com/search?q=ブラックボックステスト+ホワイトボックステスト+違い) 」です。3 種類ぐらい説明記事を読んで、内容を理解しましょう。

### ホワイトボックステストの網羅性について

ホワイトボックステストではプログラムの内部構造を意識して実施します。
したがって、すべての条件・分岐を網羅して実施するのが理想です。
しかし、全てを網羅しようとするとテスト内容が膨大になってしまい現実的ではありません。

そこで、省略できるテストを省略しつつ、バグが検出できるような方法で実施します。
条件・分岐をどれくらい細かく網羅してテストするのかを網羅性と呼びます。
