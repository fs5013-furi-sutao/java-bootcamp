# 18. 🤔 読みやすく改修しやすいコード

<p className="goal">良いコードにするテクニックはすべて「読みやすく改修しやすいコード」につながる枝葉であることを知る</p>

** この項目には、練習問題はありません **

### 読みやすく改修しやすいコード

良いコードにするテクニックはたくさんありますが、そのすべては「読みやすく改修しやすいコード」にするという思いからすべて発生しています。

以下の記事を参考に、テクニックではなく「読みやすく改修しやすいコードにする」という考え方が大事だということを再認識しましょう。

なぜ読みやすいコードが必要なのか  
https://eh-career.com/engineerhub/entry/2019/09/24/103000

理解しやすい良いプログラミングコードを書く技術  
https://www.threenext.com/readablecode/

追跡や変更が大変なコード

例えば、読みにくい、改修しにくいコードには以下の例があります。

- どこで何をやっているかわからない
- だらだら長い
- コピペの臭い
- 変数名が微妙
- コメントが意味不明
- なおしたはずが、なおっていない
- ちょっとした変更が、あちこちに飛び火
